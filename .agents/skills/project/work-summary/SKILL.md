# 工作总结（掌图 / zhangtu）

自动检测指定时间段内的项目变更，生成结构化工作总结。

## 角色定位

你将作为 **项目助理**，协助用户快速梳理掌图原型工作产出，生成可直接分享的总结。

## 核心流程

确定时间范围 → 扫描变更 → 选择格式 → 生成总结

### 步骤 1：确定时间范围

优先级：

1. **用户明确指定**  
   - 「帮我写今天的日报」→ 当日 00:00 ~ 当前  
   - 「这周的周报」→ 本周一 00:00 ~ 当前  
   - 「3 月 25 日到 3 月 28 日」→ 精确区间  
2. **未指定（默认）** → 当日日报  

**首次确认模板**（必须先发，再扫描）：

```text
总结范围：YYYY-MM-DD HH:mm ~ YYYY-MM-DD HH:mm
输出格式：精简版（默认）

如需调整范围或切换为详尽版，请告知。否则开始扫描变更。
```

### 步骤 2：扫描变更

**扫描范围（zhangtu）**：

| 路径 | 类型 | 说明 |
|------|------|------|
| `src/pages/` | 原型页面 | 业务页变更（可注明是否系统页 skills/design-system） |
| `src/requirements/` | 全局需求 | 需求 md 新增/修改 |
| `src/pages/**/spec.md` | 页面事实源 | 页面目标与区块说明 |
| `src/pages/**/zhangtu.requirements.ts` | 页面需求锚点 | 锚点注册变更 |
| `.zhangtu/iterations/` | 版本/迭代 | 迭代 JSON 变更（只读状态目录，总结中描述即可） |
| `docs/` | 项目文档 | 指南、能力地图、工作流等 |
| `src/common/` | 公共模块 | 共享组件/品牌等 |
| `.agents/skills/project/` | 项目技能 | 技能包变更 |
| `zhangtu.config.json` / `zhangtu.capabilities.json` | 配置 | 项目/能力配置 |

**扫描方式**（纯 git + 文件，无外部依赖）：

```bash
# 优先：带 commit 的变更
git log --since="<start>" --until="<end>" --name-status --pretty=format:"%H|%ai|%s"

# 补充：工作区未提交
git diff --name-only
git diff --cached --name-only

# 非 git 或需按 mtime
find <dirs> -type f -newermt "<start>" ! -newermt "<end>"
```

**分析维度**（每个变更文件）：

- 路径与类型（页面 / 需求 / 迭代 / 文档 / 技能 / 其他）  
- 变更类型（新增 / 修改 / 删除）  
- 关联 commit message（如有）  
- 关联 `spec.md` 标题或需求 id（如有）  

### 步骤 3：确定输出格式

| 格式 | 适用 | 侧重点 |
|------|------|--------|
| **精简版**（默认） | 日报、站会 | 核心产出 + 关键进展 |
| **详尽版** | 周报、交接 | 完整变更列表 + 统计 + 技术细节 |

切换口令：「用详尽版」「精简一点」等。

### 步骤 4：生成总结

**直接在对话中展示**，不默认写文件。用户要求保存时可写入 `docs/`。

#### 精简版结构

```markdown
# 工作总结（精简）YYYY-MM-DD ~ YYYY-MM-DD

## 核心产出
- …

## 页面 / 需求
- …

## 版本与文档
- …

## 备注
- （仅用户口述的进行中/待办可写在此）
```

#### 详尽版结构

```markdown
# 工作总结（详尽）YYYY-MM-DD ~ YYYY-MM-DD

## 概览统计
| 类型 | 新增 | 修改 | 删除 |
|------|------|------|------|
| 页面 | | | |
| 需求 | | | |
| 迭代 | | | |
| 文档 | | | |

## 按类型明细
### 页面 …
### 全局需求 …
### 迭代 …
### 文档与技能 …

## Commit 摘要
- …

## 备注
- …
```

## 约束

1. **只读** — 不修改项目文件（用户明确要求保存总结除外）  
2. **时间准确** — 严格按范围扫描  
3. **客观** — 仅基于文件变更与 commit，不虚构  
4. **不推测待办** — 不自动写「进行中/计划」；仅用户口述可追加  
5. **隐私** — 不输出 token、密钥、个人信息
