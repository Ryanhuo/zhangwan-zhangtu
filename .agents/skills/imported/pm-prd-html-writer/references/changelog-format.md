# Skill 修订历史书写规范

本文档规定 `pm-prd-html-writer` Skill 的 `CHANGELOG.md` 应如何记录每次修订。**Agent 每次修改本 Skill 时必须遵守。**

***

## 1. 何时必须写修订记录

以下任一情况发生时，**必须**在 `CHANGELOG.md` 顶部追加一条记录：

| 触发条件 | 示例 |
|----------|------|
| 修改 `SKILL.md` | 新增模式、调整工作流、更新检查清单 |
| 修改 `references/*` | 模板、约定、映射规则、主题 CSS |
| 修改 `scripts/*` | 新增/修改自动化脚本 |
| 修改 `evals/evals.json` | 新增评测用例 |
| 删除或重命名 Skill 内文件 | 废弃旧脚本 |

**不需要**写入本 CHANGELOG 的情况：

- 仅修改 `src/docs/` 下某份产品 PRD 交付物（应写在该 PRD 自身的修订历史表中）
- 仅修改 `src/prototypes/` 原型代码（应同步 `spec.md`）

***

## 2. 记录格式（强制）

每条修订记录须包含以下字段，**不得省略**：

```markdown
### vX.Y.Z — YYYY-MM-DD HH:mm:ss

| 字段 | 内容 |
|------|------|
| **变更类型** | 新增 / 修改 / 修复 / 废弃 |
| **修订人** | Agent / 用户名 |
| **涉及文件** | 逗号分隔的相对路径列表（相对于 `skills/pm-prd-html-writer/`） |
| **变更说明** | 详细描述（见 §3） |
| **影响范围** | 对 Agent 行为、生成产物、脚本用法的影响 |
```

### 2.1 时间格式

- **必须**精确到秒：`YYYY-MM-DD HH:mm:ss`
- 使用**本地时间**（24 小时制）
- 示例：`2026-06-11 09:44:16`
- **禁止**只写日期（如 `2026-06-11`）或模糊时间（如「今天上午」）

获取当前时间（Windows PowerShell）：

```powershell
Get-Date -Format 'yyyy-MM-dd HH:mm:ss'
```

### 2.2 版本号规则

Skill 版本采用语义化版本 `vMAJOR.MINOR.PATCH`：

| 递增 | 条件 |
|------|------|
| **MAJOR** | 破坏性变更：工作流重构、模板 DOM 结构大变、脚本接口不兼容 |
| **MINOR** | 新功能：新模式、新脚本、新 reference 文档 |
| **PATCH** | 修复与小优化：文案、样式微调、bug 修复 |

同时更新 `CHANGELOG.md` 顶部的 **当前 Skill 版本** 行。

***

## 3. 变更说明写法（详细描述要求）

变更说明**不能**只写一句话概括，须按下面结构展开：

1. **做了什么**：逐条列出具体改动（用编号 ①②③…）
2. **为什么改**：业务或技术原因（若明显可简写）
3. **怎么用**：若涉及新脚本/新参数，写出命令示例
4. **注意点**：破坏性变更、需手动同步的文件、已知限制

### 合格示例

> ① 新增 `sync-prd-theme.mjs`，可将 `prd-antd-theme.css` 注入 HTML 的 `<style>` 块；② 主色从 `#2563EB` 改为 Ant Design `#1677ff`；③ 表格表头从深色渐变改为 `#fafafa`。运行 `node skills/pm-prd-html-writer/scripts/sync-prd-theme.mjs` 同步已有 PRD。已有 PRD 不会自动更新，须手动执行同步脚本。

### 不合格示例

> 更新了 UI 主题。（过于简略，未说明改了哪些文件、如何生效）

***

## 4. 追加位置

新记录**始终**插入在 `## 修订记录（最新在上）` 标题之后、上一条记录之前：

```markdown
## 修订记录（最新在上）

### v1.5.0 — 2026-06-12 14:30:00   ← 新记录写在这里
...

### v1.4.0 — 2026-06-11 09:44:16   ← 上一条
```

**禁止**在文件末尾追加（否则不符合「最新在上」约定）。

***

## 5. Agent 自检清单

修改本 Skill 并准备结束前，确认：

- [ ] `CHANGELOG.md` 已追加本条修订记录
- [ ] 时间格式为 `YYYY-MM-DD HH:mm:ss`
- [ ] 变更说明含具体改动点（非一句话带过）
- [ ] `涉及文件` 列出所有变更路径
- [ ] `CHANGELOG.md` 顶部「当前 Skill 版本」已更新
- [ ] 若变更影响 Agent 流程，`SKILL.md` 已同步更新
