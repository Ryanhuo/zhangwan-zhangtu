# 第二章「全局说明」模板

企业后台 HTML PRD 须在 **第二章 需求分析** 中设 **`2.3 全局说明`**（或 `二、需求分析` → `2.3 全局说明`），集中描述跨模块通用规则。

> **写作原则**：全局说明在此**统一写一次**；后续各 `3.x` 功能模块若涉及相同规则，**不再重复描述**，在字段说明「说明」列或业务规则中写 **`遵循 2.3 全局说明`** 即可。

HTML 标题示例（供 TOC 解析）：

```html
<h2 id="第二章-需求分析">第二章 需求分析</h2>
<h3 id="23-全局说明">2.3 全局说明</h3>
<blockquote>
  <p>全局说明的内容，在这里统一说明。如后面功能有涉及到的部分不再描述。</p>
</blockquote>
```

## 规则说明（可复制正文）

以下为标准 **全局说明** 示例，生成 PRD 时按需整段引用或裁剪；与本产品无关的条目可删除，**勿在 3.x 模块内重复展开**。

### (1) 多语言

界面原型中带有 **【🌐】** 图标的代表多语言功能，统一使用系统现有规则。

### (2) 唯一性

字段说明列表中，「说明」列备注内容包含 **「唯一」** 的字段，属于唯一性要求，**禁止重复**。如有重复，在触发 **【保存】** 时进行重复提示（字段下方内联错误，文案如「{字段名称}已存在，请重新输入」）。

### (3) 「说明」字样字段

带 **「说明」** 字样的字段，代表对当前字段或功能的一种描述（解释说明），**没有功能及交互作用**。

### (4) 列表排序

所有列表均支持按所选字段进行 **「升序」** 或 **「降序」** 排序。

### (5) 编码类字段

所有 **编码类型** 的字段限制输入内容：支持大小写字母、数字、`_`，**禁止**输入其他类型字符。各模块字段表「限制」段可写 **`遵循 2.3 全局说明（编码规则）`**。

### (6) 权限配置

① 所有菜单均纳入 **菜单权限** 管理范围。通过权限配置，支持对角色、用户设置授权功能菜单的访问权限。

② 所有填写单据的字段 **下拉数据**，通过 **数据权限** 进行管理配置，支持对角色、用户设置授权下拉的数据访问权限。

③ **同类** 数据权限的配置均采取 **并集** 管理。

④ **非同类** 数据权限的配置均采取 **交集** 管理。

**同类数据权限推演**

| 名称 | 权限控制 | 数据权限（组 1） | 数据权限（组 2） | 显示结果 |
|------|----------|------------------|------------------|----------|
| 业务组 | 组织 | 公司 1；公司 2；公司 3 | 公司 2；公司 3；公司 4 | 公司 1；公司 2；公司 3；公司 4 |
| 张三 | 项目 | 项目 A；项目 B；项目 C | 项目 D | 项目 A；项目 B；项目 C；项目 D |

**非同类数据权限推演**

| 名称 | 权限控制 | 数据权限 | 单据数据范围 | 显示结果 |
|------|----------|----------|--------------|----------|
| 角色或用户 | 组织 | 公司 1；公司 2；公司 3 | 公司 2；公司 3；公司 4 | 公司 2；公司 3 |
| 角色或用户 | 项目 | 项目 A；项目 B；项目 C | 项目 A；项目 B；项目 D | 项目 A；项目 B |

### (7) 下拉选项通用规则

单据中所有字段属于 **下拉选项** 的，统一遵循：

① 支持 **模糊查询**，默认显示最近使用的 **3** 个数据项。

② 下拉内容默认加载 **100** 条数据；超过 **10** 条通过滚动查询，并在数据项最末尾显示 **【加载更多】** 按钮。

③ 超出部分点击 **【加载更多】** 再增加显示 **50** 条。

④ 下拉数据项带 **层级关系** 时：

1. 点击展开，显示全部同级数据项。
2. 模糊查询时，所查询的数据项后方显示上级名称，如：`销售区/销售中心/公司/集团`。

字段说明「说明」列涉及下拉数据源时，可写 **`遵循 2.3 全局说明（下拉规则）`**，并补充本字段特有数据源路径。

### (8) 分页

分页功能支持：选择上一页、下一页、跳转页、选择页面显示数量。

① 页面显示数量：**10 条/页**、**20 条/页**、**50 条/页**、**100 条/页**、**200 条/页**；**默认 20 条/页**。

② 超出页面显示范围的，滚动显示。

### (9) 列表规则

① 加载或刷新列表，默认按 **编码升序** 排序。

② 列表有新增功能时，所有新增数据按 **新增时间先后顺序升序** 排列，即 **先增加的在最上方** 显示。

③ 列表成员有层级的，**默认折叠**。

④ 带有分页的列表，显示数量根据分页选项过滤。

### (10) 禁用数据

所有 **禁用的数据项**，在 **填单界面不显示**；**查询类字段** 可以正常显示。

### (11) 维度 / 数据源下拉枚举

**源维度、目标维度、数据源** 等字段的下拉选项为：

| 序号 | 名称 |
|------|------|
| 1 | 组织 |
| 2 | 人员 |
| 3 | 岗位 |
| 4 | 职级 |
| 5 | 核算机构（单位管理） |
| 6 | 全部维度数据 |

## 与各章节的引用关系

| 场景 | 写法 |
|------|------|
| 字段说明「限制」段 | `限制：遵循 2.3 全局说明（编码规则）` 或写本字段特有规则 |
| 字段说明「说明」段（下拉） | `…；联动：无；必填提示：…；数据源遵循 2.3 全局说明（下拉规则）` |
| 3.x.5 业务规则 | 仅写**本模块特有**规则；通用排序/分页/权限写 `遵循 2.3 全局说明` |
| 3.x.5 校验规则 | 唯一性、必填提示等模块特例可写；与全局重复的不再展开 |
