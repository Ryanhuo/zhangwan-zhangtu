# HTML PRD 写作与排版约定

本文档从 **`src/docs/管理模式_PRD_V1.0.html`** 等交付物反推，定义企业后台 HTML PRD 的结构、组件与交互约定。与 `pm-prd-writer` 的 Markdown 章节一一对应，供 `pm-prd-html-writer` 动笔前必读。

> **参考实现**：`src/docs/PRD/管理模式/管理模式_PRD_V1.0.html`

***

## 0. 视觉规范（Ant Design 主题）

HTML PRD 文档 UI 参照 **Ant Design 5** 组件风格，主色为 `#1677ff`（Ant Design 蓝）。

| 设计元素 | 对应 Ant Design 组件 | 说明 |
|----------|---------------------|------|
| 侧栏目录 | Menu / Sider | 白底侧栏，选中项 `#e6f4ff` 背景 + `#1677ff` 文字 |
| 侧栏标题 | Layout Header | 纯色 `#1677ff` 顶栏 |
| 正文区 | Card | 白底卡片，圆角 8px，浅灰边框 |
| 表格 | Table | 表头 `#fafafa`，行悬停 `#fafafa` |
| 原型区 | Card + Tabs | 卡片容器 + 下划线 Tab |
| 状态标签 | Tag | 浅色底 + 彩色边框（非实心块） |
| 引用块 | Alert info | `#e6f4ff` 背景 + `#91caff` 边框 |
| 返回顶部 | FloatButton | 主色方块按钮 |

**主题源文件**：`references/prd-antd-theme.css`  
**同步脚本**：`node skills/pm-prd-html-writer/scripts/sync-prd-theme.mjs [file.html ...]`  
新建或改版 HTML PRD 后，应运行同步脚本使模板与已有交付物样式一致。

## 1. 章节结构（与 pm-prd-writer 对齐）

### 1.1 五大章骨架

| HTML 章 | 标题示例 | MD 来源 |
|---------|----------|---------|
| 一 | 项目概述 | 概述、背景、范围 |
| 二 | 需求分析 | 功能需求、流程图、**2.3 全局说明**、非功能 |
| 三 | 功能设计 | **3.1–3.x 功能模块**（核心） |
| 四 | 技术方案 | 技术方案（可标 [待研发补充]） |
| 五 | 项目计划 | 里程碑、验收、风险 |

### 1.2 第三章：按功能模块拆分（3.x）

每个功能模块使用 **同一套五段结构**，标题编号与 MD 一致：

```
3.x   [模块名称]
3.x.1 功能描述
3.x.2 界面原型          ← prototype-gallery 多状态截图 + 可选 iframe
3.x.3 字段说明          ← 8 列字段表
3.x.4 按钮说明          ← 独立按钮表
3.x.5 业务规则          ← 列表 + 颜色/状态约定
```

**标题格式要求**（供目录解析）：

- 章：`一、项目概述`、`二、需求分析` …
- 节：`3.1 顶部导航栏`、`3.5 图像增强`
- 小节：`3.5.2 界面原型`

目录脚本会从标题文本解析 `3.5.2` 等编号并单独展示，**勿在 h2/h3 中省略数字前缀**。

### 1.4 第二章「全局说明」（2.3）

跨模块通用规则须在 **第二章 `2.3 全局说明`** **集中写一次**；后续 `3.x` 涉及相同规则时 **不再重复**，在字段说明或业务规则中写 **`遵循 2.3 全局说明`** 即可。

| 项 | 约定 |
|----|------|
| **位置** | HTML：`<h3 id="23-全局说明">2.3 全局说明</h3>`；MD：`## 2.3 全局说明` |
| **开篇** | 引用块说明：「全局说明的内容，在这里统一说明。如后面功能有涉及到的部分不再描述。」 |
| **正文** | 规则说明 (1)–(11)：多语言、唯一性、说明字段、列表排序、编码、权限、下拉、分页、列表、禁用数据、维度枚举等 |
| **完整模板** | 见 [`global-rules-template.md`](global-rules-template.md)（含权限推演表、维度枚举表、HTML 嵌入示例） |
| **模块引用** | 字段「限制」/「说明」/ `3.x.5 业务规则` 写 `遵循 2.3 全局说明（编码规则）` 等，仅补充模块特例 |

企业后台 HTML PRD **默认须包含** `2.3 全局说明`；轻量 HTML 模式或用户明确裁剪时可省略无关条目，但不得在各 `3.x` 内重复粘贴全文。

### 1.5 第三章「业务规则」（3.x.5）

每个 `3.x` 模块的 **`3.x.5 业务规则`** 描述**本模块特有**行为，按 **六类** 分小节书写，**有则写、无则省略**：

| 小节 | 典型内容 |
|------|----------|
| **排序规则** | 新增项按时间升序（先增在上）等模块级排序 |
| **搜索规则** | 模糊查询、不区分大小写、回车/【查询】触发、结果按分组展开 |
| **加载规则** | 页面初始布局（如左侧折叠、右侧空态）、点击后加载详情区 |
| **交互规则** | 悬停显示/隐藏按钮、操作菜单、折叠拉伸、不可删条件 |
| **校验规则** | 本模块必填/唯一性；通用规则写「遵循 2.3 / 字段说明表」 |
| **删除约束** | 关联单据/流程时禁止删除（可并入交互规则） |

**完整模板**（含「核算架构」示例与 HTML/Markdown 片段）：见 [`business-rules-template.md`](business-rules-template.md)。

与 **2.3 全局说明** 分工：全站通用的列表排序、分页、下拉、编码字符集等 **不在 3.x.5 重复**；本页特有的树形布局、分区加载、悬停按钮等 **写在 3.x.5**。

### 1.3 字段说明表（8 列）

标准列（与 `pm-prd-writer` 企业模板一致）：

| 字段名称 | 类型 | 必填 | 可修改 | 长度 | 控件类型 | 输入类型 | 说明 |

复杂模块按 UI 区域分多张表（如 `【查询条件】`、`【基本信息】`），标题用 `<p><strong>【区域名】</strong></p>` 标注。

#### 「必填」「可修改」列：仅填「是」或「否」

| 列 | 填写规则 |
|----|----------|
| **必填** | 只允许 **`是`** 或 **`否`**，表示该字段是否必填 |
| **可修改** | 只允许 **`是`** 或 **`否`**，表示**默认/主场景**下用户能否编辑该字段 |

禁止在「可修改」列写条件句或描述性文字，例如：

- ❌ `不可修改`、`条件可修改`、`审批中不可改`、`初始可编辑`
- ❌ `是（审批中否）`、`否，只读`

若可修改性依赖**单据状态、角色、页面模式**等条件，一律写入 **「说明」列**，「可修改」列仍只填主场景下的 `是` 或 `否`。

页面内联编辑时，`prd-inline-editor.js` 会对「必填」「可修改」列做 **是/否** 校验；条件说明请写在同行的「说明」列。

#### 「控件类型」列：标准名称（枚举）

须从下列名称中**择一**填写，禁止自造同义词（如「文本框」「下拉单选」「多行文本」等旧称）：

| 分类 | 允许值 |
|------|--------|
| 日期时间 | 日期选择、日期范围选择、时间选择、时间范围选择 |
| 文本与数字 | 输入框、文本域、多行文本框、数字输入框、数字范围、密码、一次性密码、验证码 |
| 选择与枚举 | 选择器、单选、多选、多选组合、树选择、级联选择、自动补全、穿梭框 |
| 布局与展示 | 标签、选项卡、开关、滑动输入、评分、空格 |
| 地址与手机 | 组合示例地址、地区、详细地址、组合示例手机、国家、手机 |
| 媒体与附件 | 上传、图片上传、拖拽上传 |
| 其他 | 颜色选择器、提及 |

> **「空格」**：仅占位的布局控件，无数据绑定；说明列各段均写「无」即可。

#### 「说明」列：格式 / 限制 / 联动 / 必填提示（无则写「无」）

「说明」列须按字段实际情况写清四类信息；**某项不存在时，该段写 `无`**，不可省略整段：

| 段落 | 何时必填 | 写法示例 |
|------|----------|----------|
| **格式** | 控件类型为日期/时间/范围类 | `格式：YYYY-MM-DD`、`格式：HH:mm:ss`、`格式：YYYY-MM-DD ~ YYYY-MM-DD` |
| **格式** | 其他控件 | `格式：无` |
| **限制** | 有字符数、数值范围、输入条件、唯一性等 | `限制：最多 50 字符，唯一，仅字母数字`；`限制：取值 0～100` |
| **限制** | 无任何限制 | `限制：无` |
| **联动** | A 变更导致 B 更新/过滤/禁用等 | 见下方联动示例 |
| **联动** | 无联动 | `联动：无` |
| **必填提示** | 「必填」列为 **是** | 见下方「必填校验与提示」 |
| **必填提示** | 「必填」列为 **否** | `必填提示：无` |

**推荐句式**（同一单元格内用中文分号 `；` 连接）：

```
格式：…；限制：…；联动：…；必填提示：…
```

状态/角色相关的**可编辑性**、**数据源**、**展示规则**等，写在上述四段之后，用分号继续追加（无则省略该段）。

#### 必填校验与提示（全局 + 字段级）

HTML PRD 参照 **Ant Design 5 Form** 交互，与项目原型一致。

**全局规则**（写在 `3.x.5 业务规则` → **校验规则**，模块内统一说明一次即可）：

| 项 | 约定 |
|----|------|
| **必填标识** | 标签旁红色 `*`（必填字段） |
| **校验触发** | 点击【保存】【提交】【确认】等**提交类按钮**时统一校验；失焦校验仅在业务规则中特别声明时启用 |
| **提示位置** | **字段下方**内联红色文案（Form help），控件边框高亮为错误态 |
| **不用顶部提示** | 单项「未填必填项」**不得**仅用页面顶部 Message / Toast；顶部仅用于保存成功、网络异常等**全局反馈** |
| **多项未填** | 所有未通过字段**同时**在各自下方展示错误；页面**滚动定位至第一个**错误字段 |

**字段级「必填提示」段**（「必填」= 是 时必填）：

写法：`必填提示：字段下方，{提示语}`

提示语按 **输入类型 / 控件类型** 选用下列模板（`{字段名称}` 替换为当前行字段名）：

| 场景 | 提示语模板 | 示例 |
|------|------------|------|
| 手工输入（输入框、文本域、多行文本框、数字输入框、数字范围、密码、一次性密码等） | `请输入{字段名称}` | `必填提示：字段下方，请输入编码` |
| 验证码 | `请填写验证码` | `必填提示：字段下方，请填写验证码` |
| 选择（选择器、单选、多选、多选组合、树选择、级联选择、日期选择、日期范围选择、时间选择、时间范围选择、穿梭框、自动补全、地区等） | `请选择{字段名称}` | `必填提示：字段下方，请选择生效日期` |
| 上传（上传、图片上传、拖拽上传） | `请上传{字段名称}` | `必填提示：字段下方，请上传附件` |
| 开关、评分等需「开启/完成」的控件 | `请完成{字段名称}` | `必填提示：字段下方，请完成风险确认` |
| 自定义文案 | 写完整提示语 | `必填提示：字段下方，请至少填写一条续报内容` |

**格式/规则类错误**（非「未填」）写在「限制」段或 `3.x.5 校验规则`，与必填提示区分，例如：

- 长度超限 → 字段下方：`最多输入 50 个字符`
- 格式错误 → 字段下方：`请输入正确的手机号码`
- 唯一性冲突 → 字段下方：`编码已存在，请重新输入`

<tr>
  <td>生效日期</td><td>日期</td><td>是</td><td>是</td><td>-</td>
  <td>日期选择</td><td>选择</td>
  <td>格式：YYYY-MM-DD；限制：无；联动：无；必填提示：字段下方，请选择生效日期</td>

### 2.1 多状态截图（prototype-gallery）

多状态模块（弹窗、加载、成功/失败、只读等）**每种 UI 状态各 1 张截图**，用网格展示：

```html
<div class="prototype-gallery">
  <figure class="prototype-figure">
    <img src="./assets/screenshots/management-mode/35-modal-generating.png"

    <figcaption>图 3.5-1：生成中</figcaption>
  </figure>
  <figure class="prototype-figure">
    <img src="./assets/screenshots/management-mode/35-modal-success.png"
         alt="图像增强-成功" class="screenshot">
    <figcaption>图 3.5-2：增强成功</figcaption>
  </figure>
</div>
```

| 模块类型 | 截图数量 |
|----------|----------|
| 单一布局（顶栏、侧栏） | ≥ 1 张 |
| 多状态（增强、处置流程） | **每个 distinct UI 状态 1 张** |

批量截取：

```bash
node skills/pm-prd-html-writer/scripts/capture-prototype-states.mjs \
  --url http://127.0.0.1:51720/prototypes/management-mode/
```

输出：`src/docs/assets/screenshots/<prototype-name>/`，命名建议 `3x-状态.png`（如 `35-modal-success.png`）。

### 2.2 双模式区（prototype-section）与全屏交互

交互原型**不得**直接用固定高度 iframe 内嵌（宽屏原型如 1280px 会被裁切右侧）。须使用 **预览缩放 + 点击全屏**：

```html
<div class="prototype-iframe-preview" data-design-width="1280" data-design-height="800">
  <div class="prototype-iframe-hint">点击全屏交互</div>
  <div class="prototype-iframe-scale">
    <iframe class="interactive-prototype interactive-prototype--preview"
            src="./assets/prototypes/management-mode/index.html"
            title="管理模式 - 交互原型"
            sandbox="allow-scripts allow-same-origin allow-forms"
            loading="lazy">
    </iframe>
  </div>
  <div class="prototype-iframe-placeholder" hidden>全屏交互中，关闭全屏后恢复此处预览</div>
</div>
```

页面底部须有 `#prototypeFullscreen` 全屏层（见 `html-template.html`）。

| 行为 | 说明 |
|------|------|
| **预览区** | 按 `data-design-width/height` 等比缩放，展示完整布局（含右侧区域） |
| **点击预览** | iframe 移入全屏层，100% 视口可交互 |
| **退出** | 右上角 × 或 `Esc`，iframe 回到预览区且不丢状态 |

`data-design-width` 应与原型 `style.css` 的 `min-width` 一致（如 management-mode 为 1280）。

全局或模块级可保留 **静态 Tab + 交互 Tab** 的 `prototype-section` 结构。

| 场景 | iframe `src` | 说明 |
|------|-------------|------|
| **离线交付（默认）** | `./assets/prototypes/<name>/index.html` | 用 `export-prototype-for-prd.mjs` 导出 |
| 仓库内在线评审 | `/prototypes/<name>/` | 需 `npm run dev` |
| 不可用 | 隐藏交互 Tab | 不得留 `about:blank` |

导出离线原型：

```bash
node skills/pm-prd-html-writer/scripts/export-prototype-for-prd.mjs \
  --prototype management-mode \
  --prd src/docs/管理模式_PRD_V1.0.html
```

### 2.3 截图可点击放大（Lightbox）

- 点击 lightbox 内图片不关闭；仅点击遮罩、`×` 或 `Esc` 关闭

## 3. 颜色与状态标签

字段表、业务规则中涉及 **色值 / 处置状态** 时，使用带背景的标签，而非纯文本 `#ff4d4f`：
