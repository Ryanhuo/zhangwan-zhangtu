# Skills 目录说明

本目录在文件侧已经做了来源隔离，实际维护规则如下：

- 项目自带技能放在 `./project/`
- 用户导入技能放在 `./imported/`
- 顶层同名目录仅作为兼容别名，不是第二份真实内容

这套约定的目的：

- 方便项目内长期维护“随仓库分发”的技能
- 方便区分“用户个人导入”的技能资产
- 同时兼容依然从 `.agents/skills/<skill-id>/SKILL.md` 读取技能的旧工具链

如果要新增项目内置技能，请优先放到 `./project/<skill-id>/`。
如果是页面上传或用户手动导入的技能，请放到 `./imported/<skill-id>/`。
