数值区间输入框，对应真实的 `InputNumberRange`——两个用"~"连接的输入框
（最小值/最大值），带一个尾部清除图标，用于数值类筛选字段（价格区间、花费
区间、播放量区间等）。

```jsx
<InputNumberRange from={min} to={max} onChange={(f, t) => { setMin(f); setMax(t); }} />
```

用法：默认宽度 240px，与 Input/Select/DatePicker 共用同一套筛选行尺寸规范。
两个输入框共享同一个聚焦边框状态（任一侧聚焦，整个容器边框变绿），不是各自
独立的边框——这是与两个独立 Input 并排的关键区别，视觉上是"一个控件"而不是
"两个控件"。`onChange` 每次都携带完整的 `(from, to)` 两个值，即使只改动了
其中一侧。

用法限制：不做数值范围校验（不检查 from ≤ to），也不限制只能输入数字——
如果业务需要这些校验，应在 `onChange` 回调里自行处理，不要期望组件拦截
非法输入。

边界情况：`clearable` 默认开启，只要任一侧有值就显示清除图标，点击会同时
清空两侧；只需要清空单侧时应直接编辑对应输入框，没有单独清除按钮。
