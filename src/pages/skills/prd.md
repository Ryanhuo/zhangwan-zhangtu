# 技能管理 PRD

## 页面背景与目标

掌图原型系统内置技能库，支持产品经理在本地项目管理一组 Claude Code 技能（skill），技能作为可复用的 AI 能力单元被页面发现、导入、预览、删除。当前页面面向两类用户场景：

- **项目初始化者**：查看项目自带技能清单，确认框架能力覆盖范围。
- **日常使用 PM**：浏览已导入技能、导入新技能、删除不再需要的用户技能。

技能数据通过 `/api/skills` 读取本地 `.agents/skills/` 目录，不连接远程仓库，所有操作在本地完成。

## 核心功能清单

### 技能列表

- 左侧技能目录按「项目自带」与「用户导入」两个分组展示，外加「框架运维」独立分组。
- 每个技能条目显示名称，点击后在右侧区域展示完整 Markdown 详情。
- 列表顶部显示技能总数（项目自带 + 用户导入 + 框架运维）。

### 能力分层（tier）

- 技能分为两个层级：**能力技能**（capability）和**运维技能**（ops）。
- 能力技能按「项目自带」/「用户导入」分组，默认展开。
- 运维技能收进「框架运维」分组，**默认折叠**，避免干扰日常 PM 工作流。
- 分层判定由服务端 `OPS_SKILL_SLUGS` 集合控制（当前：`zhangtu-installer`、`zhangtu-init-prototype-project`、`protolink`）。

### 技能导入

- 支持上传 `.md`、`.zip`、`.skill` 三种格式。
- 导入弹窗内拖拽或点击上传，文件需包含根级 `SKILL.md` 且 frontmatter 含 `name` 与 `description`。
- 导入成功后自动注册到 Claude Code 技能目录（`~/.claude/skills/`），可被 CLI 直接调用。
- 导入时校验 `name` 唯一性，与已有技能重名时拒绝并提示。

### 技能详情

- 右侧展示选中技能的 Markdown 正文（strip frontmatter 后渲染）。
- 详情区顶部显示分类标签（项目自带 / 用户导入）、调用名、文件路径。
- 调用名冲突时以 warning 提示用户修改 `SKILL.md` 的 `name` 字段。

### 技能删除

- 仅用户导入技能可删除，项目自带技能不可删除。
- 删除前弹出确认对话框，操作不可恢复。

### 侧栏折叠

- 左侧栏支持收起/展开，状态持久化到 `sessionStorage`，单次会话内保持。

## 关键交互说明

| 交互 | 触发方式 | 行为 |
|------|---------|------|
| 浏览技能列表 | 页面加载 | 自动请求 `/api/skills` 并渲染分组列表 |
| 查看技能详情 | 点击左侧技能条目 | 请求 `/api/skills/:id/detail`，右侧渲染 Markdown |
| 导入技能 | 点击「导入技能」按钮 | 弹出 Modal，拖拽或点击上传文件 |
| 删除技能 | 点击用户导入技能行的删除图标 | 确认弹窗 → 调用 `/api/skills/:id/delete` → 刷新列表 |
| 折叠/展开分组 | 点击分组标题 | 切换该分组内技能列表的可见性 |
| 收起/展开侧栏 | 点击侧栏切换按钮 | 切换左侧栏宽度，状态写入 sessionStorage |

## 验收标准

1. **技能列表正常加载**：页面打开后 `/api/skills` 返回数据，左侧三组（项目自带、用户导入、框架运维）正确渲染，运维分组默认折叠。
2. **技能详情可查看**：点击任意技能条目，右侧在 1 秒内展示该技能的 Markdown 正文，元信息（分类标签、调用名、路径）正确。
3. **技能导入成功**：上传有效 `.md` / `.zip` / `.skill` 文件后，技能出现在用户导入分组，消息提示成功；上传无效文件或重名技能时，提示明确错误信息。
4. **技能删除成功**：删除用户导入技能后，该项从列表移除，选中状态重置；刷新页面后列表持久一致。
5. **侧栏折叠持久化**：收起侧栏后刷新页面，侧栏状态保持（sessionStorage）；new tab 中状态独立。
