# 掌图原型工具 — 同事安装指南

掌图（zhangtu）是一套本地原型管理工具，支持页面预览、需求标注、版本管理和分享链接发布，全部在本机运行，无需联网账号。

---

## 前置条件

安装前确认以下软件已就绪：

| 软件 | 最低版本 | 检测命令 |
|------|---------|---------|
| Node.js | 18.x 以上 | `node -v` |
| Git | 任意版本 | `git --version` |

Node.js 官网下载：https://nodejs.org（选 LTS 版本）

---

## 安装步骤

### 第一步：创建工作区

在你想存放原型项目的目录下，运行以下命令（将 `我的项目` 替换为你的项目名，只用英文和连字符）：

```bash
# 进入你想放项目的目录（例如桌面）
cd ~/Desktop

# 从 GitHub 安装掌图系统（只做一次）
npm install -g zhangwan-zhangtu   # 如果已经安装过可跳过

# 初始化一个新的原型工作区
npx zhangtu init 我的项目
```

> 如果系统管理员已将 `zhangwan-zhangtu` 放在了内网或指定路径，按他们给的安装命令操作即可。

---

### 第二步：安装依赖

```bash
cd 我的项目
npm install
```

等待安装完成（首次大约 1–2 分钟）。

---

### 第三步：启动预览

```bash
npm start
```

浏览器会自动打开预览页面（端口 6320）。如果没有自动打开，手动访问：

```
http://127.0.0.1:6320
```

---

## 基本使用

### 添加页面

在 `src/pages/` 下新建文件夹，按以下结构添加文件：

```
src/pages/我的页面/
├── index.html      # 必须
├── index.tsx       # 必须
└── spec.md         # 可选，页面说明（第一行 # 标题会作为页面名）
```

保存后预览 Shell 会自动刷新，左侧导航出现新页面。

### 创建版本

在预览 Shell 右上角点击「版本管理」→「+ 新建版本」，填写版本名和说明，勾选要包含的页面，保存即可。

### 预览版本

在版本列表点击「预览」，新标签页打开只读版本预览，可分享给团队评审。

## 常见问题

**Q: 端口被占用了怎么办？**
```bash
npm start -- --port 6321
```

**Q: 页面没出现在导航栏？**
运行 `npm run check:pages` 查看具体诊断信息。

**Q: 如何拿到分享链接？**
需要先配置发布 Token（联系系统管理员获取），然后在版本管理里点「发布」。

## 目录说明

```
我的项目/
├── src/pages/          ← 在这里添加你的原型页面
├── src/resources/      ← 放需求文档、参考资料
├── zhangtu.config.json ← 项目名称和主题配置
└── .zhangtu/           ← 工具自动生成，不要手动修改
```
