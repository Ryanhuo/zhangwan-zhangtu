# 掌图个人操作手册

本文档记录两类工作的操作方式：管理同事的原型项目，以及维护掌图系统本身。

---

## 两个文件夹的分工

| 文件夹 | 用途 | 路径 |
|--------|------|------|
| **zhangwan-zhangtu/** | 系统代码——Shell、CLI、模板、rules、skills | `~/Desktop/zhangwan-zhangtu/` |
| **各项目工作区** | 同事原型内容——页面、版本、spec 文档 | 独立目录，由 `init` 创建 |

两个文件夹**互不干扰**：系统代码不存原型内容，项目工作区不改系统逻辑。

---

## 一、本地开发管理

### 为同事初始化新工作区

在 `zhangwan-zhangtu/` 目录内运行：

```bash
# 在当前目录创建新项目工作区（建议放在系统目录同级）
cd ~/Desktop/zhangwan-zhangtu
npm run zhangtu -- init ../项目名称
```

或者直接在目标父目录操作：

```bash
cd ~/Desktop
npx --prefix ./zhangwan-zhangtu zhangtu init 项目名称
```

工作区创建后结构如下：

```
~/Desktop/
├── zhangwan-zhangtu/   ← 系统代码（不动）
└── 项目名称/           ← 新工作区（给同事用）
    ├── src/pages/
    ├── zhangtu.config.json
    └── package.json
```

### 启动某个工作区进行开发

```bash
cd ~/Desktop/项目名称
npm install        # 首次或系统更新后执行
npm start          # 启动掌图预览 Shell，浏览器自动打开
```

页面文件保存后自动热更新，无需重启。

### 在现有工作区新增页面

在工作区的 `src/pages/` 下按以下结构新建目录，保存后 Shell 自动发现：

```
src/pages/新页面/
├── index.html             # <div id="root"> + <script src="./index.tsx">
├── index.tsx              # createRoot(...).render(...)
├── spec.md                # 页面说明，# 标题作为页面名
├── styles/page.css        # 页面样式（引用该工作区设计系统的令牌）
└── zhangtu.requirements.ts  # 需求模块（可选）
```

新页面必须使用工作区约定的设计系统（本仓库自身是 zhangwan-design），先查看对应 `.agents/skills/project/` 下的设计令牌再动笔。

## 二、代码维护

系统代码在 `zhangwan-zhangtu/` 目录，以下是常见维护场景：

### 维护 Shell（预览界面）

Shell 是 `scripts/zhangtu/shell.html` 单文件，修改后刷新浏览器即生效。

常见改动：
- 侧边栏样式、按钮行为 → 在 `<style>` 或 `<script>` 中修改
- 版本卡片、弹窗布局 → 找对应 `.version-*` class
- API 请求逻辑 → 找 `preview-server.mjs`

### 维护 CLI / 后端逻辑

```
scripts/zhangtu/
├── cli.mjs           # 命令分发与参数解析
├── preview-server.mjs # HTTP API 路由与业务逻辑
├── discovery.mjs     # 页面发现
├── iterations.mjs    # 版本 CRUD
└── proto-hub.mjs     # 发布组件
```

改页面识别规则时，`discovery.mjs`、`vite.config.ts`、`scripts/check-pages.mjs` 三处同步修改。

### 维护 Rules（AI 指引）

| 文件 | 说明 |
|------|------|
| `CLAUDE.md` | Claude Code 主要指令（架构、设计规范、命令） |
| `AGENTS.md` | 代码 Agent 指引（目录结构、API、开发注意事项） |
| `docs/` | 补充说明文档（theme-guide、resource-guide 等） |

修改后提交即生效，不需要额外步骤。

### 维护 Skills（AI 技能）

Skills 目录：`.agents/skills/project/zhangwanUI/`

常见改动：
- 设计令牌更新 → 修改 `css.json` 和 `colors_and_type.css`
- 组件规范更新 → 修改 `components/*.json` 和 `components.css`
- 组件预览页更新 → 修改 `preview/*.html`

Skills 也会随系统代码打包发布，同事更新后自动获得最新版本。

## 三、上传说明（发布系统更新）

### 前提：配置 GitHub 仓库

如果 `package.json` 中还有 `GITHUB_OWNER` 占位符，先替换：

```bash
# 查看当前状态
grep GITHUB_OWNER package.json

# 替换为实际用户名（一次性操作）
sed -i '' 's/GITHUB_OWNER/你的GitHub用户名/g' package.json
```

确认远端仓库已添加：

```bash
git remote -v
# 若为空，添加：
git remote add origin https://github.com/你的用户名/zhangwan-zhangtu.git
```

---

### 日常更新流程

```bash
cd ~/Desktop/zhangwan-zhangtu

# 1. 确认改动无误（至少运行以下检查）
npm run check:pages
npm run typecheck
npm run build

# 2. 提交代码
git add -A
git commit -m "feat: 简短描述本次改动"

# 3. 推送到 GitHub
git push origin main
```

---

### 同事如何获取更新

同事在其工作区目录运行：

```bash
npm update zhangwan-zhangtu
npm install
```

Shell、CLI、Skills、模板均会同步到最新版本。页面内容（`src/pages/`）是同事自己的，不受影响。

---

### 注意事项

- **不要把 `.zhangtu/` 提交进系统仓库**——这是运行时状态，已在 `.gitignore` 中排除。
- **不要把同事的工作区代码（原型页面）提交进 `zhangwan-zhangtu/`**——两个目录完全分离。
- Skills 文件（`.agents/skills/`）属于系统代码，应随系统仓库一起提交和分发。
- `publish-config.json`（含 token）已在 `.gitignore` 中，不会被误提交。
